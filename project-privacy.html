Anonymization and Deanonymization of MovieLens Data

Project Poster

Project Paper

About the Project

My group and I sought to find the most effective techniques to anonymize a dataset, and we then subsequently tested an algorithm to deanonymize the same dataset. Our approach involved utilizing a primary dataset from LetterBoxd and supplementing it with an auxiliary dataset from MovieLens, where both datasets contain user data, movie information, and ratings. In terms of anonymization, Data Generalization ended up being the most effective technique of the techniques we chose to test. In terms of deanonymization, we found that when implementing the Scoreboard_RH function that was developed by Arvind Narayanan and Vitaly Shatiko, we were able to find a few potential matches across the datasets with high certainty, showing proof of concept in the effectiveness of the algorithm.

Anonymization & Deanonymization Steps

1. We anonymized the MovieLens dataset using data generalization via binning, adding synthetic data, and performing data swapping.

2. This anonymization performed very well, as we could not deanonymize the data when all three anonymization techniques were used.

3. We then attempted to deanonymize only the binned data using Narayanan and Shmatikov's Scoreboard_RH function (this would take approx. 220 years to run).

4. Next, we reduced the dataset to just 100 users to deanonymize (approx. 17 years).

5. We additionally filtered to only match using the 10,000 least common films, which resulted in no matches.

6. We then tried using only the 100 most common films, which resulted in no users being filtered out.

7. Finally, after relaxing the Scoreboard_RH function and reducing the dataset to only five users, we were able to match all five across the datasets.

Our conclusion is that when a large, sparse dataset is appropriately anonymized, it is extremely difficult to deanonymize.
